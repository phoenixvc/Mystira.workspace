# Fix: Invalid Agent ID Format

## The Problem

Your agents were created with **custom IDs** instead of **OpenAI-style auto-generated IDs**.

### Current Situation (WRONG):
```
ID: asst-mystira-write-v01   ← Custom ID with hyphens
ID: asst-mystira-judge-v01   ← Custom ID with hyphens
ID: mystira-rubric-v01       ← Custom ID, no asst prefix
ID: mystira-refiner-v01      ← Custom ID, no asst prefix
```

### Required Format (CORRECT):
```
ID: asst_abc123xyz456        ← OpenAI format: asst_ + random chars
ID: asst_def789uvw012        ← Underscore, not hyphen
ID: asst_ghi345rst678        ← Auto-generated by Azure
ID: asst_jkl901mno234        ← Cannot be custom
```

## Why This Fails

The Azure AI Agents service uses the **OpenAI Assistants API** under the hood, which strictly requires IDs in the format `asst_[random]`. Custom IDs like `asst-mystira-write-v01` are rejected with:

```
Error 404: No assistant found with id 'asst-mystira-write-v01'
```

Even though the agent exists, the API can't use it because of the invalid ID format.

## The Fix

You need to **recreate the agents** with proper auto-generated IDs.

### Step 1: Delete Existing Agents (Optional)

You can either:
- **Keep them** for reference (they won't interfere)
- **Delete them** from Azure Portal:
  1. Go to https://portal.azure.com
  2. Navigate to your AI Foundry project
  3. Click "Agents" → Select each agent → Delete

### Step 2: Create New Agents with Proper IDs

This will:

2. Configure them with file_search tools

## Why Custom IDs Don't Work

The Azure AI Agents SDK uses this flow:

1. Your code → Azure AI Agents SDK
2. Azure AI Agents SDK → Azure OpenAI Service
3. Azure OpenAI validates the assistant ID format
4. **FAILS** if ID doesn't match `^asst_[a-zA-Z0-9]+$`

When you created agents with custom IDs, Azure stored them but the OpenAI API can't retrieve them because they fail validation.

## Prevention

**Never manually specify agent IDs.** Always let the SDK auto-generate them:

❌ **WRONG:**
```csharp
// Don't do this
var agent = client.CreateAgent(
    id: "my-custom-agent-id",  // ← BAD!
    model: "gpt-4",
    name: "My Agent"
);
```

✓ **CORRECT:**
```csharp
// Let Azure generate the ID
var agent = client.Agents.CreateAgent(
    model: "gpt-4",
    name: "My Agent"  // Name can be anything, ID is auto-generated
);

Console.WriteLine(agent.Value.Id);  // Will be asst_abc123xyz...
```

## Summary

| Issue | Cause | Fix |
|-------|-------|-----|
| 404 Error | Custom agent IDs don't match OpenAI format | Recreate agents with auto-generated IDs |
| Invalid ID format | Used hyphens or no prefix | Use SDK-generated IDs (asst_random) |
| Config has wrong IDs | Copied names instead of IDs | Copy auto-generated IDs from create output |
